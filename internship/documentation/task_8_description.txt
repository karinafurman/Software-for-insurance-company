В ходе выполнения данного  задания, я познакомилась с файлом gitignore, лежащим в корне проекта.
Файл gitignore нужен для того, чтобы в общий репозиторий кода не попадали файлы, которые привязаны
к моему локальному компу; в нем прописываются правила, какие файлы должны игнорироваться и не попадать
в github [информация с лекции].
В файл gitignore рекомендуется добавлять: компилированные файлы, пакетные директории, файлы и директории IDE,
файлы системы управления версиями, файлы журналов и временные файлы, локальные конфигурационные файлы,
файлы зависимостей, файлы, автоматически генерируемые документами.
Добавление этих файлов в gitignore помогает поддерживать репозиторий чистым,
избегать конфликтов, упрощать совместную работу в команде.
